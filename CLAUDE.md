# CLAUDE.md

이 저장소는 코드 프로젝트가 아니라 옵시디언 Vault다. 다루는 대상은 마크다운 노트.

## 구조

| 폴더 | 용도 |
|---|---|
| `00_Inbox/` | 미분류 노트. 새 노트의 기본 생성 위치 |
| `10_Daily/` | 데일리 노트. 파일명 `YYYY-MM-DD.md` 고정 |
| `20_Projects/` | 프로젝트별 노트. `20_Projects/<프로젝트명>.md` 또는 하위 폴더 |
| `30_TIL/` | 오늘 배운 것. 파일명은 주제 그대로 (예: `Docker 볼륨 권한.md`) |
| `40_Troubleshooting/` | 문제 해결 기록 |
| `90_Templates/` | 노트 템플릿. 새 유형의 노트를 만들 땐 여기부터 확인 |
| `99_Attachments/` | 첨부파일. 직접 건드릴 일 없음 |

## 노트 작성 규칙

- 새 노트는 해당 유형의 템플릿(`90_Templates/`) 구조를 그대로 따른다. `{{date}}`는 `YYYY-MM-DD`, `{{title}}`은 파일명으로 치환.
- frontmatter의 `tags`는 소문자 kebab-case, `created`는 `YYYY-MM-DD`.
- 파일명에 `: / \ # ^ [ ] |` 사용 금지 (옵시디언 링크 문법과 충돌).
- 노트 간 연결은 위키링크 `[[노트명]]`. 마크다운 링크는 외부 URL에만 사용.

## 자주 하는 요청과 처리 방법

- **"오늘 배운 거 TIL로 정리해줘"** → `30_TIL/`에 TIL 템플릿으로 생성하고, 오늘 데일리 노트(`10_Daily/YYYY-MM-DD.md`)의 "오늘 배운 것" 섹션에 `[[링크]]`를 추가한다. 데일리 노트가 없으면 Daily 템플릿으로 먼저 만든다.
- **"이 에러 트러블슈팅으로 남겨줘"** → `40_Troubleshooting/`에 Troubleshooting 템플릿으로 생성. 증상 섹션에 에러 메시지 원문을 코드블록으로 넣는다 (나중에 검색되도록). 해결됐으면 `resolved: true`.
- **"인박스 정리해줘"** → `00_Inbox/`의 노트를 내용을 보고 적절한 폴더로 옮기고 frontmatter를 보강한다. 어디로 보낼지 애매한 노트는 옮기지 말고 목록으로 보고.
- **"프로젝트 노트 만들어줘"** → `20_Projects/`에 Project 템플릿으로. 이후 진행 상황 업데이트 요청은 그 노트의 "로그" 섹션에 날짜 소제목으로 추가한다 (기존 로그는 수정하지 않음).

## 하지 말 것

- 노트 삭제 금지. 필요하면 이동하고 보고한다.
- 요청받지 않은 노트 본문 수정 금지. 오타 교정도 요청이 있을 때만.
- `.obsidian/workspace.json`은 개인 상태 파일 — 수정도 커밋도 하지 않는다.

## 커밋

- 메시지는 한국어 한 줄로, 무엇이 바뀌었는지만 (예: `TIL 2건 추가, 인박스 정리`).
- 노트 커밋 요청 시 `.obsidian/` 변경분은 빼고 노트만 묶는다 (설정 변경을 명시적으로 요청한 경우 제외).
